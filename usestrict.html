JavaScript Strict Mode Notes

Definition

Strict mode is a way to run JavaScript in a "stricter" version that catches errors, prevents unsafe behaviors, and disables some confusing features.

Enabled with:

"use strict";

Enabling Strict Mode

  • Whole script: Put "use strict"; at the top of the file.
  • Function only: Put "use strict"; at the start of the function.
// Applies to whole script
"use strict";

// Applies only inside this function
function myFunc() {
    "use strict";
}

Key Features / Effects

  1. No accidental globals 
    "use strict";
x = 10; // ❌ ReferenceError
  2. Prevents assignment to read-only properties 
    "use strict";
const obj = {};
Object.defineProperty(obj, "prop", { writable: false });
obj.prop = 5; // ❌ TypeError
  3. Disallows duplicate parameter names 
    "use strict";
function sum(a, a) {} // ❌ SyntaxError
  4. Changes this in functions

    Normal mode: this → global object

    Strict mode: this → undefined

    "use strict";
function f() { console.log(this); }
f(); // undefined
  5. with statement is forbidden 
    "use strict";
with (obj) { } // ❌ SyntaxError
  6. eval and arguments are restricted

    Cannot assign to eval or arguments; arguments won’t reflect parameter changes.

  7. Strict mode is applied by default in ES6 modules and classes

Comparison Table

Feature Non-Strict Mode Strict Mode
Undeclared variable Allowed (creates global) ReferenceError
Duplicate parameters Allowed SyntaxError
Assign read-only property Fails silently TypeError
this in function Global object undefined
with statement Allowed SyntaxError
Interview-Friendly One-Liner:

"use strict" enforces stricter parsing and error handling, catching silent errors and preventing unsafe practices like accidental globals.
